## Learned
- VL53L5CX uses `LPn` (Low Power Not) pin for I2C address differentiation — same function as XSHUT on older ST sensors, just renamed
- Address reassignment via I2C is not persistent across power cycles — must re-run init sequence every boot
- Sensor uploads ~86KB of firmware over I2C at every boot; setting I2C bus to 1MHz reduces startup from ~9s to ~1.4s
- At 45° mount, reported slant distance requires `× cos(45°) = × 0.7071` correction to get true perpendicular distance
- VL53L5CX has a 45°×45° FoV; at 500mm slant range all zones are valid and ground crosstalk immunity holds (>60cm threshold)

## Friction
- VL53L5CX I2C address assignment requires strict 100ms boot delay after LPn goes HIGH — skipping it causes `StartRanging` status 255 failure
- Library firmware upload is blocking and slow at default 100kHz; must explicitly set freq=1_000_000
